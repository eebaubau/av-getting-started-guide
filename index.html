Chapter 1
Download & Install
Before anything else, you need the Claude desktop app on your computer.
1
Download Claude Desktop
  1. Open your browser and go to claude.ai/download
  2. Click the download button for your operating system (Mac or Windows)
  3. Open the downloaded file and follow the installation prompts
  4. Once installed, open the Claude app — you should see a chat interface
2
Sign In to Your Account
  1. Use your Anthropic account to sign in
  2. Username: startups@angelesvc.com
  3. Password: A code is sent to Outlook — check your inbox
Claude has two modes: Chat (regular conversation) and Cowork (files + tools + skills). This guide uses Cowork.
3
Install the Claude Chrome Extension

This extension lets Claude open, click, and fill out web pages for you — useful for skills that need to interact with web apps (LinkedIn research, Google Docs uploads, Airtable tweaks through the UI, etc.). Works the same on Mac and Windows.

  1. Open Google Chrome. If you don't have Chrome, install it from google.com/chrome first.
  2. Go to the Chrome Web Store: chromewebstore.google.com/detail/claude-for-chrome
    (Or search the Chrome Web Store for "Claude for Chrome" and pick the one by Anthropic.)
  3. Click "Add to Chrome" → "Add extension" in the popup that appears.
  4. Pin the extension to your toolbar: click the puzzle icon in the top-right corner of Chrome, find Claude, and click the pin icon next to it.
  5. Click the Claude icon in your toolbar and sign in with startups@angelesvc.com (same account as Claude Desktop).
The extension is optional but strongly recommended — some skills work much better (or only work) when Claude can drive Chrome directly.
One-time
System Setup
Before you can run skills like one-pager, investment-memo, or pitchbook-sourcing, your computer needs Python, Node.js, and poppler installed. Good news: Claude Code will do it all for you.
1
Open Claude Code

In Claude Desktop, look at the top of the screen for the mode tabs and click Claude Code (next to Chat and Cowork). Claude Code is the mode that can run commands on your computer — that's what we need to install things.

2
Paste this prompt

Copy the whole block below and paste it into Claude Code:

Please set up my machine to run the Angeles Ventures skills. Detect whether I'm on macOS or Windows, then install: - Python 3 (and make sure it's on PATH) - Node.js LTS (and make sure it's on PATH) - poppler (on Windows, put it at C:\poppler and add C:\poppler\Library\bin to my user PATH) Then install these Python packages with pip: python-pptx, openpyxl, pdf2image, pillow, numpy, pymupdf And this Node package globally with npm: docx When everything is done, verify the install by running: python3 --version (or python --version on Windows) node --version pdftoppm -h If anything fails, explain what went wrong in plain English and suggest the fix. Do not ask me to run commands myself unless something truly requires admin input.

Claude will walk through the install, ask for any permissions it needs, and tell you when it's done.

3
Restart Claude Desktop

Fully quit Claude Desktop (Cmd+Q on Mac, right-click the taskbar icon → Quit on Windows) and reopen it. This lets Cowork pick up the newly installed Python and Node.

Why Claude Code and not Cowork? Cowork can read files and use connectors but cannot execute shell commands on your machine. Claude Code can. Setup is a one-time thing; after this you mostly live in Cowork.
Prefer to do it manually? Click for step-by-step commands →

macOS

Install Homebrew first (if you don't already have it):

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Then:

brew install python node poppler pip3 install python-pptx openpyxl pdf2image pillow numpy pymupdf --break-system-packages npm install -g docx

Verify:

python3 --version node --version pdftoppm -h

Windows

  1. Install Python 3 from python.org/downloads — check "Add Python to PATH" on the first screen.
  2. Install Node LTS from nodejs.org/en/download (accept defaults).
  3. Install poppler: open PowerShell and paste the one-liner below. It downloads, extracts, and PATH-configures poppler automatically.
    $r=Invoke-RestMethod "https://api.github.com/repos/oschwartz10612/poppler-windows/releases/latest";$a=($r.assets|?{$_.name -like "Release-*.zip"})[0];$z="$env:TEMP\poppler.zip";Invoke-WebRequest $a.browser_download_url -OutFile $z;if(Test-Path "C:\poppler"){Remove-Item "C:\poppler" -Recurse -Force};Expand-Archive $z -DestinationPath "C:\" -Force;Get-ChildItem "C:\" -Filter "poppler-*" -Directory|Select -First 1|Rename-Item -NewName "poppler";$p=[Environment]::GetEnvironmentVariable("Path","User");if($p -notlike "*C:\poppler\Library\bin*"){[Environment]::SetEnvironmentVariable("Path","$p;C:\poppler\Library\bin","User")};Write-Host "Done."
    PowerShell command failed? Install poppler by clicking (no code) →
    1. Download the zip. Open your web browser and go to github.com/oschwartz10612/poppler-windows/releases. Under the first release at the top (the one that says "Latest"), find the file called Release-something.zip and click it to download. It'll land in your Downloads folder.
    2. Unzip it. Open File Explorer (the yellow folder icon in your taskbar), go to Downloads, right-click the zip you just downloaded, and choose Extract All.... In the window that pops up, change the destination to just C:\ (delete what's there and type those 3 characters), then click Extract.
    3. Rename the folder. In File Explorer, click "This PC" in the left sidebar, then double-click "Local Disk (C:)". You'll see a folder called something like poppler-25.12.0. Right-click it, choose Rename, and change the name to just poppler (delete the version number and the dash). Press Enter.
    4. Verify the path exists. Double-click into poppler, then into Library, then into bin. You should see a file called pdftoppm.exe. If you see it, you're good. If not, something went wrong above — try again.
    5. Tell Windows where it is (the PATH part).
      1. Press the Windows key (the one with the Windows logo, bottom-left of your keyboard).
      2. Type the word environment (doesn't matter if the Start menu search is weird, just type it).
      3. Click the result called "Edit environment variables for your account" (NOT the one that says "for your system" — the account one).
      4. A window pops up with two lists. In the top list, find the row called Path, click it once to select it, then click the Edit... button on the right.
      5. Another window pops up with a list of folder paths. Click the New button on the right.
      6. A blank row appears. Type exactly: C:\poppler\Library\bin and press Enter.
      7. Click OK on this window. Click OK on the previous window too. Close any other windows that popped up.
    6. Close PowerShell completely and open it again (press Windows key, type PowerShell, press Enter). Then type pdftoppm -h and press Enter. If you see a wall of text starting with "pdftoppm version", poppler is working.
  4. Close and reopen PowerShell, then run:
    pip install python-pptx openpyxl pdf2image pillow numpy pymupdf npm install -g docx
  5. Verify:
    python --version node --version pdftoppm -h
Common errors and what to do

Never used Terminal or PowerShell before? Don't panic. "Terminal" (Mac) and "PowerShell" (Windows) are apps where you type commands instead of clicking. You open them, paste the command, press Enter. That's it. You don't need to know any of the language.

"PowerShell says: running scripts is disabled on this system"
Your company's Windows is locked down. Paste this command first, press Enter, type Y to confirm, then try again:

Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

"python is not recognized" or "command not found: python"
Python installed but isn't "on PATH". Windows: re-run the Python installer from python.org, but this time check the box "Add Python to PATH" on the very first screen. Mac: at the end of the Homebrew install, it printed two lines starting with eval. You have to copy those two lines, paste them in Terminal, press Enter. That's what "adding to PATH" means.

"ModuleNotFoundError: No module named 'pptx'" (or pdf2image, numpy, etc.)
The pip install step didn't finish, or it installed into a different Python than the one Claude is using. Mac: re-run the pip command with --break-system-packages at the end. Windows: re-run the pip command in a fresh PowerShell window.

"Cowork can't run Python / skill fails with 'no python found'"
You installed Python but Claude Desktop is still using the old PATH it loaded when it started. Fully quit Claude Desktop (Cmd+Q on Mac, right-click taskbar icon → Quit on Windows) and open it again.

Windows installer shows a "Do you want to allow this app to make changes?" popup
That's UAC (Windows Account Control). Click Yes. Claude Code cannot click that popup for you — it needs a human.

Chapter 2
Switch to Cowork
Cowork mode lets Claude read your files, connect to tools, and run skills. It's the powerhouse.
3
Open Cowork Mode
  1. In the Claude Desktop app, look at the top of the screen for Chat and Cowork tabs
  2. Click Cowork to switch into that mode
  3. You'll see a new interface where Claude can access files and run tasks
4
Connect Your OneDrive Folder

This is how Claude gets access to the Angeles Ventures shared workspace files.

  1. Install the OneDrive app on your computer (not the browser — the actual desktop app). Download it from onedrive.com.
  2. Sign in with your startups@angelesvc.com email.
  3. Open this link in your browser:
    Angeles Venture Group on SharePoint
  4. Look for "Add shortcut to My files" — usually near the top, sometimes under a "..." menu. Click it.
  5. Wait a minute. Then open Finder on your Mac. In the left sidebar, look for OneDrive with "Angeles Venture Group" inside. If you see the numbered folders (00. Decks, 01. Fundraising...), you're good!
  6. Now open Claude Desktop → Cowork → click the folder icon → navigate to that folder
  7. Claude now has read and write access to all the fund files!
How to verify: Open Finder → left sidebar → OneDrive → "Angeles Venture Group" → you should see all the numbered folders.
You only need to select the folder once per session. When you start a new Cowork session, you may need to select it again.
Chapter 3
Connect Airtable
MCP is the bridge between Claude and external apps. Airtable is the most important one for deal flow.
5
Add the Airtable Connector
  1. In Claude Desktop, go to Settings (gear icon, bottom-left)
  2. Look for Connectors or Integrations
  3. Search for "Airtable" and click Connect
  4. You'll be redirected to Airtable — sign in and authorize access
  5. Once done, you'll see Airtable listed as connected in Settings

Chapter 5
Build Your First Skill
Let's create "New Company Pre-Screening" — folders, website research, LinkedIn lookup, and Airtable record.
7
Create the Skill File

In Cowork mode, tell Claude:

Create a new file called new-company-pre-screening.md in 03. Funding/1. Skills/

Or create the file manually in OneDrive. The important thing: it must be a .md file in the right folder.

8
Write the Skill Contents

Every skill follows a simple structure. Here's the template — fill in the header, then write your own steps:

new-company-pre-screening.md
--- name: new-company-pre-screening description: "Sets up a new company for diligence: creates folder structure in OneDrive, researches company website and founder LinkedIn, creates Airtable record. Triggers: 'pre-screening of [Company]', 'new company pre-screening', 'add company to pipeline to pre-screen'." --- # New Company Pre-Screening ## What this skill does Sets up a new company for diligence: creates folder structure, researches website + LinkedIn, creates Airtable record for pre-screening purposes. ## How to run Type: "Pre-screening for [Company Name]" Must ask for the website URL if user hasn't provided it. ## Inputs - Company name (required) - Company website URL (required) ## Steps [Write your steps here! Think about what Claude should do and in what order. For example: - Should it check if the company already exists? - What folders should it create? - What info should it extract from the website? - Where should it save the results? Be as specific as possible. The more detail you give Claude, the better the output.] ## Outputs - Company folder with all subfolders - Airtable record with populated fields

Your turn!

Write the specific steps you want the agent to follow. Think through the workflow, test it, and iterate. That's how skills get built!

9
Save the File

Make sure the file is saved as:

03. Funding / 1. Skills / new-company-pre-screening.md

Since this is in OneDrive, it automatically syncs to everyone on the team.

Help
Troubleshooting

Claude doesn't see my files

Make sure you selected the OneDrive folder in Cowork. Look for the folder icon and re-select it. Verify the folder appears in Finder first.

Airtable isn't connected

Go to Settings → Connectors and check if Airtable is listed. If not, click it and re-authorize.

Claude used the wrong skill

Be explicit: "Read 03. Funding/1. Skills/new-company-pre-screening.md and run it for [Company]"

Mistake in the Airtable record

If you wrote to Total Deal Flow (production) by accident, go to Airtable and delete or fix the record directly. To avoid this, always default to EduTest (sandbox) when testing — production writes require explicit confirmation.

I ran out of credits

The Startups account runs on a Pro plan, which has tighter usage limits. For heavier skills (investment memos, logo scraping, long research runs), a Max plan may be needed. If you hit the limit mid-run, pause and ping Eduardo before upgrading.
